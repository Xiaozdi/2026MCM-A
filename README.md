## 项目简介

本项目围绕智能手机电池耗电问题，建立了一套**面向状态演化的连续时间数学模型**，以刻画电池剩余电量（State of Charge, SOC）在不同使用场景下随时间变化的动态过程。与经验回归或黑箱预测方法不同，该模型强调**物理可解释性**：一方面利用电池等效电路描述放电过程中的内在电化学行为，另一方面将整机功耗拆分为多个可解释的硬件子模块，从而实现对耗电来源与续航差异的定量分析。该建模思路与论文中提出的 “physically grounded continuous-time differential framework” 保持一致。

从论文框架看，模型主要包含四个层次：  
1. **SOC 连续时间演化方程**：以微分方程形式描述电量变化；  
2. **电化学动力学子模型**：采用 Thevenin/Shepherd 型等效电路，引入开路电压、极化电压和内阻等变量；  
3. **硬件级功耗分解模型**：将总功耗拆分为屏幕、CPU、网络、GPS、后台任务与基础功耗等部分；  
4. **扩展耦合因子**：考虑环境温度与电池老化对有效容量和放电过程的影响。

当前仓库保留的代码主要展示了上述框架中的**功耗建模、场景仿真与结果可视化部分**，并以 **Moto G60** 参数进行示例化整理，用于说明模型思路、程序组织方式及典型结果生成流程。

## 方法概述

### 1. SOC 动态演化

论文中，电池放电过程被表示为一个连续时间动力系统。SOC 的变化率由瞬时功率消耗、电池端电压和有效容量共同决定，并通过数值方法进行离散求解。论文进一步将该过程与开路电压、极化电压以及放电电流联系起来，形成完整的状态更新系统。

在当前保留版本中，可将其简化理解为：

dSOC/dt = - P_total(t) / C
其中：

- `SOC`：电池剩余电量百分比
- `P_total(t)`：时刻 `t` 的总功耗
- `C`：电池等效总能量容量

> 需要说明的是，上式仅是仓库展示层面的简化表达；论文中的完整模型还进一步引入了端电压、放电电流、温度修正与老化修正等耦合因素。

### 2. 电化学动力学基础

论文的核心优势之一，在于并未将电池视为单纯的能量黑箱，而是采用 **Thevenin/Shepherd Equivalent Circuit** 对内部状态进行刻画。模型中引入：

- 开路电压 `U_oc(SOC)`
- 极化电压 `U_p(t)`
- 内阻 `R_int`
- 放电电流 `I(t)`

并通过电压—电流—功率关系，将电化学状态与整机功耗连接起来。论文还进一步引入温度方程与有效容量修正项，用于分析环境温度与老化效应对放电过程的影响。

### 3. 总功耗分解

根据论文的建模结构，总功耗由六个子模块组成：

P_total = P_base + P_screen + P_cpu + P_network + P_gps + P_background

这一“白箱式”功耗分解方法是全文的重要基础，它使得不同使用场景下的续航差异可以被具体归因到各个硬件模块，而不仅仅停留在总体耗电层面。论文在模型建立与功耗分解分析部分均采用了这一结构。

## 功耗子模块说明

### 1. 基础功耗 `P_base`

表示设备在维持系统基本运行时产生的静态功耗，包括主板漏电、必要传感器与基本后台维持过程等。

### 2. 屏幕功耗 `P_screen`

论文中将屏幕视为关键耗电子系统，并依据 OLED 自发光特性建立内容相关功耗模型。其能耗与亮度、白色像素比例以及刷新率有关，因此在网页浏览、视频播放与高刷新率游戏场景中通常占据主导地位。

### 3. CPU 功耗 `P_cpu`

CPU 功耗由静态功耗与动态功耗构成。论文将其与核心频率、利用率、多核结构以及温度泄漏效应相关联，以体现高负载场景下处理器能耗的快速增长。

### 4. 网络功耗 `P_network`

网络模块用于描述 WiFi、4G、5G 等无线通信方式在不同活动强度下的功耗差异。论文指出，导航等场景中的额外耗电，并非全部来自 GPS，本质上还受到持续网络数据传输的显著影响。

### 5. GPS 功耗 `P_gps`

GPS 功耗在模型中通常以定位请求频率或使用强度建模。论文的分解结果显示，GPS 本身的直接功耗占比并不高，导航场景下更主要的耗电来源往往是屏幕常亮与网络连接。

### 6. 后台任务功耗 `P_background`

后台任务用于表征系统同步、应用维持、定时唤醒等非前台交互过程。论文分析表明，该模块在待机场景下相对重要，但在高强度前台活动中占比会明显下降。

## 典型研究内容对应关系

根据论文内容，完整项目主要围绕以下四类问题展开：

### 1. SOC 动态仿真

在给定初始电量和场景输入的情况下，利用连续时间模型模拟电池电量随时间的衰减轨迹。论文采用显式 Euler 方法对微分方程组进行数值求解。

### 2. Time-to-Empty 预测

在网页浏览、视频播放、游戏、导航等场景下，预测设备从不同初始电量出发到耗尽所需的时间，并与外部参考数据进行比较。论文以 Xiaomi Redmi Note 10 Pro 为参考设备，结合标准化测试数据进行了验证。

### 3. 功耗分解分析

通过对子模块功耗进行分解，识别导致快速耗电的主要驱动因素。论文结论表明：常规任务中屏幕通常是主要功耗来源，而高性能游戏则由于高刷新率与高计算负载叠加而形成极端高耗电场景。

### 4. 敏感性分析与策略建议

论文对容量、老化、温度等关键参数进行了 One-at-a-Time (OAT) 扰动分析，发现电池老化对续航的影响显著大于环境温度波动，并据此提出了面向用户和系统设计者的优化建议。

## 主要结论

结合论文分析框架与当前保留下来的实现思路，可以概括出以下几点主要认识：

1. **模型应以连续时间动力学为核心**，而不是仅依赖经验平均值；
2. **总功耗分解是解释续航差异的关键**，屏幕、CPU、网络和后台活动共同决定场景差异；
3. **高性能游戏是最强耗电场景之一**，其本质是高刷新率显示与高计算负载的耦合；
4. **导航耗电并不等同于 GPS 耗电**，屏幕与网络才是更主要的驱动因素；
5. **电池老化比环境温度更显著地影响续航**，在高负载条件下尤为明显。
